# データ分析戦略ドキュメント

## 1. 背景

### 1.1 ビジネス目標

DTIアフィリエイトを活用した収益化サイトを構築するにあたり、**最適な戦略を選択する**ための意思決定が必要である。

### 1.2 戦略オプション

現在、以下の4つの戦略オプションが検討されている：

| オプション | コンセプト | 想定ターゲット |
|-----------|-----------|---------------|
| A案 | 初心者向け安全ガイドサイト | 有料サイト未経験者 |
| B案 | 女優特化ポータル | 特定女優のファン |
| C案 | ジャンル別比較サイト | 特定ジャンル愛好者 |
| D案 | 現在の機能強化を優先 | 既存ユーザー |

### 1.3 なぜデータ分析が必要か

戦略選択には以下の判断材料が必要：

```
┌─────────────────────────────────────────────────────────┐
│                    戦略選択の判断軸                      │
├─────────────────────────────────────────────────────────┤
│  1. 自社データ資産   → どのデータが豊富か               │
│  2. 市場機会        → どのニーズが大きいか              │
│  3. 競合状況        → どこなら勝てるか                  │
│  4. リソース制約    → 何が実現可能か                    │
└─────────────────────────────────────────────────────────┘
```

**データなき意思決定はギャンブルである。**
分析により、成功確率の高い戦略を選択する。

### 1.4 期待される成果

- 最適な戦略オプションの特定
- 具体的な実装優先順位の決定
- 収益予測の精度向上
- リスクの可視化と対策立案

## 3. 内部分析（自社データ資産）

### 3.1 分析項目

#### 3.1.1 サイト別データ量分析

**目的**: どのサイトのデータが豊富か把握し、強みを特定する

**分析SQL**:
```sql
-- サイト別動画数と割合
SELECT
    site_name,
    COUNT(*) as video_count,
    ROUND(COUNT(*) * 100.0 / (SELECT COUNT(*) FROM videos), 1) as percentage
FROM videos
GROUP BY site_name
ORDER BY video_count DESC;
```

**判断基準**:
| データ量 | 判断 |
|---------|------|
| 1万件以上 | そのサイト特化が有効 |
| 1000-1万件 | 組み合わせ戦略で活用 |
| 1000件未満 | 補助的な位置づけ |

#### 3.1.2 女優データ充実度分析

**目的**: 女優特化戦略（B案）の実現可能性を評価する

**分析SQL**:
```sql
-- 女優別動画数（上位50名）
SELECT
    actress,
    COUNT(*) as video_count,
    COUNT(DISTINCT site_name) as site_count
FROM videos
WHERE actress IS NOT NULL AND actress != ''
GROUP BY actress
ORDER BY video_count DESC
LIMIT 50;

-- 女優データの充実度
SELECT
    CASE
        WHEN actress IS NULL OR actress = '' THEN 'なし'
        ELSE 'あり'
    END as has_actress,
    COUNT(*) as count,
    ROUND(COUNT(*) * 100.0 / (SELECT COUNT(*) FROM videos), 1) as percentage
FROM videos
GROUP BY has_actress;
```

**判断基準**:
| 女優データ率 | 判断 |
|-------------|------|
| 80%以上 | 女優特化戦略が有効 |
| 50-80% | 部分的に活用可能 |
| 50%未満 | 女優特化は困難 |

#### 3.1.3 公開日分布分析

**目的**: データの鮮度を評価し、「最新情報」訴求の可否を判断する

**分析SQL**:
```sql
-- 年別公開数
SELECT
    substr(release_date, 1, 4) as year,
    COUNT(*) as video_count
FROM videos
WHERE release_date IS NOT NULL
GROUP BY year
ORDER BY year DESC;

-- 直近3ヶ月のデータ量
SELECT COUNT(*) as recent_videos
FROM videos
WHERE release_date >= date('now', '-3 months');
```

**判断基準**:
| 直近3ヶ月データ | 判断 |
|----------------|------|
| 500件以上 | 最新情報サイトとして訴求可能 |
| 100-500件 | 定期更新で補完必要 |
| 100件未満 | 過去アーカイブ型サイトが適切 |

#### 3.1.4 データ品質分析

**目的**: コンテンツとして活用可能なデータの割合を把握する

**分析SQL**:
```sql
-- 各フィールドの充実度
SELECT
    'title' as field,
    SUM(CASE WHEN title IS NOT NULL AND title != '' THEN 1 ELSE 0 END) as filled,
    COUNT(*) as total,
    ROUND(SUM(CASE WHEN title IS NOT NULL AND title != '' THEN 1 ELSE 0 END) * 100.0 / COUNT(*), 1) as fill_rate
FROM videos
UNION ALL
SELECT 'actress', SUM(CASE WHEN actress IS NOT NULL AND actress != '' THEN 1 ELSE 0 END), COUNT(*), ROUND(SUM(CASE WHEN actress IS NOT NULL AND actress != '' THEN 1 ELSE 0 END) * 100.0 / COUNT(*), 1) FROM videos
UNION ALL
SELECT 'description', SUM(CASE WHEN description IS NOT NULL AND description != '' THEN 1 ELSE 0 END), COUNT(*), ROUND(SUM(CASE WHEN description IS NOT NULL AND description != '' THEN 1 ELSE 0 END) * 100.0 / COUNT(*), 1) FROM videos
UNION ALL
SELECT 'sample_url', SUM(CASE WHEN sample_url IS NOT NULL AND sample_url != '' THEN 1 ELSE 0 END), COUNT(*), ROUND(SUM(CASE WHEN sample_url IS NOT NULL AND sample_url != '' THEN 1 ELSE 0 END) * 100.0 / COUNT(*), 1) FROM videos
UNION ALL
SELECT 'aff_link', SUM(CASE WHEN aff_link IS NOT NULL AND aff_link != '' THEN 1 ELSE 0 END), COUNT(*), ROUND(SUM(CASE WHEN aff_link IS NOT NULL AND aff_link != '' THEN 1 ELSE 0 END) * 100.0 / COUNT(*), 1) FROM videos;
```

**必須条件**: aff_linkの充実率が95%以上であること

### 3.2 分析実行スクリプト

```javascript
// scripts/analyze-data.js
const Database = require('better-sqlite3');
const db = new Database('data/data.db', { readonly: true });

function analyzeData() {
    const results = {};

    // 1. 総データ量
    results.total = db.prepare('SELECT COUNT(*) as count FROM videos').get();

    // 2. サイト別分布
    results.bySite = db.prepare(`
        SELECT site_name, COUNT(*) as count,
               ROUND(COUNT(*) * 100.0 / (SELECT COUNT(*) FROM videos), 1) as pct
        FROM videos GROUP BY site_name ORDER BY count DESC
    `).all();

    // 3. 女優データ充実度
    results.actressStats = db.prepare(`
        SELECT
            SUM(CASE WHEN actress != '' THEN 1 ELSE 0 END) as with_actress,
            COUNT(*) as total
        FROM videos
    `).get();

    // 4. 公開日分布
    results.byYear = db.prepare(`
        SELECT substr(release_date, 1, 4) as year, COUNT(*) as count
        FROM videos WHERE release_date IS NOT NULL
        GROUP BY year ORDER BY year DESC
    `).all();

    // 5. データ品質
    results.quality = db.prepare(`
        SELECT
            SUM(CASE WHEN aff_link != '' THEN 1 ELSE 0 END) * 100.0 / COUNT(*) as aff_link_rate,
            SUM(CASE WHEN title != '' THEN 1 ELSE 0 END) * 100.0 / COUNT(*) as title_rate,
            SUM(CASE WHEN actress != '' THEN 1 ELSE 0 END) * 100.0 / COUNT(*) as actress_rate
        FROM videos
    `).get();

    return results;
}

console.log(JSON.stringify(analyzeData(), null, 2));
```
